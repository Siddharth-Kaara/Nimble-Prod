DOCUMENTATION


NIMBLE is a fully automated test framework designed to streamline testing processes and enhance efficiency in automation suite development and test suite execution. The fundamental ideas that drive this product are:

Ease of Use

Manual testers can easily transition to automation testing without a steep technology learning curve.

WYSWYW (What You See is What You Write)

Testers can write scripts based on visible text on the screen, making test creation intuitive and straightforward.

Key Features

Modular Structure

The framework is built with a modular architecture for easy scalability and maintenance.

Customizable Test Cases

Allows to create custom test cases tailored to specific project requirements.

Cross-Platform Support

Supports testing across multiple platforms (Android/iOS) to ensure comprehensive test coverage.

Integration Capabilities

Easily integrates with popular CI/CD tools for seamless automation.

Reporting and Analytics

Provides detailed reports and analytics to track test results and performance metrics.

Generate

Supported Platform(s): WEB, MOBILE

Syntax:

generate "<regex>" and save it as "<variable>"

Description: Generates a random value that matches a given regular expression (regex) and stores it in the cache under a specified variable name.

Examples:

1. Generate a random code with format abcDE123:

generate "[a-zA-Z]{5}[0-9]{3}" and save it as "randomCode"

2. Generate a serial number:

generate "\d{4}-\d{4}-[A-Z]{3}" and save it as "randomSerialNumber"

3. Generate a random date that fits the format YYYY-MM-DD:

generate "[1-2][0-9]{3}-[0-1][0-9]-[0-3][0-9]" and save it as "randomDate"

4. Generate a random date that fits the format YYYY-MM-DD, such as 2023-09-15 or 1999-01-31:

generate "[1-2][0-9]{3}-[0-1][0-9]-[0-3][0-9]" and save it as "randomDate"

Enter

Supported Platform(s): WEB, MOBILE

Syntax:

enter <[variable <cached_variable>]|[<string>]> [into [(2nd|3rd)] <string>] [with exact phrase]

Description: Simulates entering a specific value into a specified field (denoted by the string position such as "2nd" or "3rd"). The value can either be a cached variable or a literal string. It optionally includes the requirement for an exact phrase match within the input field.

Parameters:

Parameter Description
variable <cached_variable> A previously stored or cached variable to be entered into the field.
<string> A literal string value to be entered into the field.
into [(2nd|3rd)] <string> Specifies the target field where the value should be entered.

• Ordinal indicator (2nd, 3rd, etc.): Optional - distinguishes between multiple fields with the same label.
• <string>: Identifies the field by its visible text, label, or placeholder.
with exact phrase Optional parameter that enforces an exact, case-sensitive match when identifying the target field.

Examples:

1. Enter a string into a field with "email" in its visible text:

enter "test@example.com" into "email"

2. Enter a previously stored variable into a specific field:

enter variable randomEmail into "Email Address"

3. Enter text into the second field that contains "password":

enter "securePassword123" into 2nd "password"

4. Enter text with exact field matching:

enter "John Doe" into "Full Name" with exact phrase

Click

Supported Platform(s): WEB, MOBILE

Syntax:

click [on] [(2nd|3rd)] <string> [with exact phrase]

Description: Simulates a click action on a specified element. The element can be identified by its visible text or label, with an optional ordinal indicator (2nd, 3rd, etc.) to distinguish between multiple elements with the same text.

Examples:

1. Click on a button labeled "Submit":

click "Submit"

2. Click on the second instance of a "Delete" button:

click on 2nd "Delete"

3. Click on an element with the exact text "Log In" (case-sensitive):

click "Log In" with exact phrase

Scroll

Supported Platform(s): WEB, MOBILE

Syntax:

scroll [up|down|left|right] [in <string>]

Description: Simulates scrolling in a specified direction within a container or on the entire page. The optional container can be identified by its visible text or label.

Examples:

1. Scroll down on the page:

scroll down

2. Scroll up in a specific container:

scroll up in "Results List"

3. Scroll right in a carousel:

scroll right in "Image Gallery"

Swipe

Supported Platform(s): MOBILE

Syntax:

swipe [up|down|left|right] [on <string>]

Description: Simulates a swipe gesture in a specified direction, optionally on a specific element identified by its text or label. This command is primarily used for mobile testing.

Examples:

1. Swipe left on the screen:

swipe left

2. Swipe up on a specific element:

swipe up on "Photo Gallery"

Slide

Supported Platform(s): WEB, MOBILE

Syntax:

slide <string> to <value>

Description: Adjusts a slider control to a specified value. The slider is identified by its label or nearby text.

Examples:

1. Set a volume slider to 75%:

slide "Volume" to 75

2. Adjust brightness slider to maximum:

slide "Brightness" to 100

relaunch the app

Examples:

1. Relaunch the app to reset its state:

relaunch the app

Restore the App to the foreground

restore the app to the foreground

Description: Brings the application back to the foreground if it was running in the background.

Examples:

1. Restore the app after checking a notification:

restore the app to the foreground

go back

Examples:

1. Navigate back to the previous screen:

go back

Go Home

Supported Platform(s): MOBILE

Syntax:

go home

Description: Simulates pressing the home button on a mobile device, minimizing the application.

1. Minimize the app by going to the home screen:

go home

Read Value

Supported Platform(s): WEB, MOBILE

Syntax:

read value of [(2nd|3rd)] <string> [and save it as <variable>]

Description: Reads the text or value from a specified element and optionally stores it in a variable for later use.

Examples:

1. Read the value of a price label:

read value of "Total Price" and save it as "price"

2. Read the value of the second error message:

read value of 2nd "Error" and save it as "errorMessage"

Wait

Supported Platform(s): WEB, MOBILE

Syntax:

wait [for] [<seconds> seconds|<string> to (appear|disappear)]

Description: Pauses test execution either for a specified number of seconds or until a specific element appears or disappears from the screen.

Examples:

1. Wait for 5 seconds:

wait for 5 seconds

2. Wait for a loading indicator to disappear:

wait for "Loading..." to disappear

3. Wait for a success message to appear:

wait for "Success" to appear

Check

Supported Platform(s): WEB, MOBILE

Syntax:

check [if] [<string> (exists|does not exist|contains <text>|equals <text>)]

Description: Verifies that a specified condition is true, such as the existence of an element or the content of an element matching expected text.

Examples:

1. Check if an element exists:

check if "Submit Button" exists

2. Check if an element contains specific text:

check if "Error Message" contains "Invalid input"

3. Check if an element's text exactly matches expected value:

check if "Total" equals "$99.99"

API Commands

Supported Platform(s): WEB, MOBILE

Syntax:

send (GET|POST|PUT|DELETE) request to "<url>" [with body <json>] [and save response as <variable>]

Description: Sends HTTP requests to specified endpoints, optionally with a JSON body, and can save the response for later use in the test.

Examples:

1. Send a GET request:

send GET request to "https://api.example.com/users" and save response as "users"

2. Send a POST request with a JSON body:

send POST request to "https://api.example.com/login" with body {\"username\": \"testuser\", \"password\": \"password123\"} and save response as "loginResponse"

Conditional Executions

Supported Platform(s): WEB, MOBILE

Syntax:

if <condition> then <commands> [else <commands>] endif

Description: Executes different commands based on whether a specified condition is true or false, allowing for branching logic in test scripts.

Examples:

1. Basic conditional execution:

if "Error Message" exists then read value of "Error Message" and save it as "errorText" else click "Continue" endif

2. Conditional execution with variable comparison:

if variable "status" equals "success" then click "Complete Order" else click "Retry" endif

Database Commands

Supported Platform(s): WEB, MOBILE

Syntax:

execute query "<sql_query>" on database "<connection_string>" [and save result as <variable>]

Description: Executes SQL queries against a specified database and optionally stores the results for use in subsequent test steps.

Examples:

1. Execute a SELECT query and save the results:

execute query "SELECT * FROM users WHERE email='test@example.com'" on database "jdbc:mysql://localhost:3306/testdb" and save result as "userData"

2. Execute an UPDATE query:

execute query "UPDATE products SET stock=100 WHERE id=1234" on database "jdbc:mysql://localhost:3306/inventory"

Kafka Eventing Commands

Supported Platform(s): WEB, MOBILE

Syntax:

send message "<message>" to kafka topic "<topic>" [with key "<key>"]

Description: Publishes messages to Kafka topics, optionally with a specified key, to test event-driven architectures and message processing.

Examples:

1. Send a simple message to a Kafka topic:

send message "{\"event\": \"user_login\", \"userId\": \"12345\"}" to kafka topic "user-events"

2. Send a message with a specific key:

send message "{\"orderId\": \"ORD-9876\", \"status\": \"shipped\"}" to kafka topic "order-updates" with key "ORD-9876"

String Operations

Supported Platform(s): WEB, MOBILE

Syntax:

perform <operation> on <string> [and save result as <variable>]

Description: Performs various string manipulation operations such as substring extraction, case conversion, or pattern matching, and optionally saves the result.

Examples:

1. Extract a substring:

perform substring from 0 to 5 on variable "fullText" and save result as "shortText"

2. Convert to uppercase:

perform uppercase on variable "username" and save result as "uppercaseUsername"

3. Extract using regex:

perform extract pattern "[0-9]+" on variable "orderConfirmation" and save result as "orderNumber"

Date Operations

Supported Platform(s): WEB, MOBILE

Syntax:

calculate date [<operation>] [and save result as <variable>]

Description: Performs date calculations such as adding days to the current date, formatting dates, or comparing date values.

Examples:

1. Get tomorrow's date:

calculate date "current + 1 day" in format "MM/dd/yyyy" and save result as "tomorrow"

2. Get the first day of next month:

calculate date "current month + 1 month, day 1" in format "yyyy-MM-dd" and save result as "nextMonthStart"

User-Defined Subroutines

Supported Platform(s): WEB, MOBILE

Syntax:

define subroutine "<name>" <commands> end subroutine call subroutine "<name>" [with parameters <param_list>]

Description: Allows the creation and execution of reusable blocks of commands, optionally with parameters, to promote code reuse and maintainability.

Examples:

1. Define and call a simple subroutine:

define subroutine "login" enter "testuser" into "Username" enter "password123" into "Password" click "Login" wait for "Dashboard" to appear end subroutine call subroutine "login"

2. Define and call a parameterized subroutine:

define subroutine "search_product" enter parameter "product_name" into "Search" click "Search Button" wait for "Results" to appear end subroutine call subroutine "search_product" with parameters {\"product_name\": \"Smartphone\"}

System Variables

Supported Platform(s): WEB, MOBILE

Description: System variables are predefined variables that provide access to system information and configuration settings.

Available System Variables:

  • $DEVICE_NAME: The name of the current device being tested.
  • $PLATFORM: The platform of the current device (Android, iOS, Web).
  • $OS_VERSION: The operating system version of the current device.
  • $SCREEN_SIZE: The screen resolution of the current device.
  • $CURRENT_DATE: The current date in YYYY-MM-DD format.
  • $CURRENT_TIME: The current time in HH:MM:SS format.

Examples:

1. Use the current date in a test:

enter $CURRENT_DATE into "Date Field"

2. Check the platform in a conditional statement:

if $PLATFORM equals "Android" then click "Android Specific Button" else click "iOS Button" endif
